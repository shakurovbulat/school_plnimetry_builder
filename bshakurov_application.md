**[ НАЗВАНИЕ ПРОЕКТА ]**

"Построитель чертежей для школьной планиметрии"

**[ ПРОБЛЕМНОЕ ПОЛЕ ]**

При изучении планиметрии в школе учащиеся сталкиваются с трудностями визуализации геометрических задач. Существующие решения либо слишком сложны для школьников (GeoGebra, AutoCAD), либо не поддерживают автоматическое построение по текстовому описанию. Учителя математики тратят значительное время на создание чертежей для уроков и контрольных работ. Отсутствует специализированный инструмент, который понимает условия задач из школьных учебников и автоматически генерирует корректные чертежи с возможностью их последующего редактирования и экспорта.

**[ ЗАКАЗЧИК / ПОТЕНЦИАЛЬНАЯ АУДИТОРИЯ ]**

• Учащиеся 7-11 классов, изучающие планиметрию
• Учителя математики в школах и лицеях
• Репетиторы по математике
• Студенты педагогических вузов

**[ АППАРАТНЫЕ / ПРОГРАММНЫЕ ТРЕБОВАНИЯ ]**

• Веб-версия: современные браузеры (Chrome 90+, Firefox 88+, Safari 14+)
• Android: версия 8.0 и выше, 100 Мб свободного места
• iOS: версия 14 и выше, 100 Мб свободного места
• Windows 10+: 200 Мб свободного места

**[ ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ ]**

• Автоматическое построение чертежей по текстовому описанию задачи
• Интерактивное редактирование созданных чертежей
• Библиотека типовых геометрических фигур и построений
• Экспорт чертежей в форматы PNG, SVG, PDF
• Сохранение чертежей в облачное хранилище
• Генерация вариативных задач на основе одного чертежа
• Подсветка геометрических элементов при наведении

**[ ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ ]**

• GeoGebra: сложный интерфейс для школьников, нет автоматического построения по описанию
• "Живая математика": платный, устаревший интерфейс
• MathCAD: ориентирован на инженерные расчеты, избыточный функционал для школ
• Онлайн-калькуляторы геометрии: ограниченный набор фигур, нет интерактивности

**[ ИНСТРУМЕНТЫ РАЗРАБОТКИ ]**

• Frontend: React.js + TypeScript
• Графика: SVG.js / Canvas API
• Мобильные приложения: React Native
• Бэкенд: Node.js + Express (для облачного сохранения)
• База данных: MongoDB / PostgreSQL
• Распознавание текста: NLP-библиотеки (TensorFlow.js)

**[ ВОЗМОЖНЫЕ РИСКИ ]**

• Сложность обработки неоднозначных формулировок условий
• Недостаточная производительность при сложных построениях
• Ошибки в геометрических алгоритмах построения

• Сложность адаптации интерфейса для мобильных устройств
